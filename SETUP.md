# Bedtime Tunes — multi-source player: setup & deploy

A two-pane player: a 1,200-track playlist on the right (read from D1), a player on the
left that picks the best source per track — **MP3 (B2) → YouTube → Spotify 30s preview**.
Contributors add tracks through a Cloudflare-Access-protected `/add` page.

```
/                       Pages root (the site)
  index.html            the two-pane player
  BT.png
/worker                 the Worker (audio.bedtimetunes.com)
  wrangler.toml
  src/index.js          /api/tracks, /api/sync, /api/add, /add page, mp3 proxy
/db
  seed.sql              generated by build_seed.py
/scripts
  build_seed.py         MP3 archive + legacy SQL + Spotify CSV -> one reordered seed
  upload.ps1            push MP3s to B2 + sync mp3_key
  resolve_youtube.py    fill youtube_id for non-owned tracks (one-time, batched)
```

## 1. Backblaze B2
Separate account (DMCA isolation). Bucket `bedtimetunes-audio`, files **Public**.
**Default Encryption: enable. Object Lock: disable.** Note the S3 endpoint + create an app key.

## 2. Build the seed (MP3 + legacy SQL + Spotify CSV)
```bash
python3 scripts/build_seed.py --csv export.csv --songs songs --sql dump.sql --out db/seed.sql
```
Owned MP3s become ids 1..146 (Tom Waits = 1) and gain a `spotify_id` where matched; the rest
of the ~1,200 become ids 147+ ordered by added date. Each row carries `mp3_key / spotify_id /
youtube_id`, genre, duration, uploader (mapped: you=2, pit=4, others default to you).

## 3. Create + seed D1
```bash
cd worker
npx wrangler d1 create bedtimetunes        # paste database_id into wrangler.toml
npx wrangler d1 execute bedtimetunes --remote --file ../db/seed.sql
```

## 4. Deploy the Worker
Edit `wrangler.toml` (database_id, B2 endpoint/bucket, CONTRIBUTORS email→id map).
```bash
npx wrangler secret put SYNC_TOKEN
npx wrangler deploy
```
Add custom domain `audio.bedtimetunes.com` (dashboard → the Worker → Domains & Routes).

## 5. Cloudflare Access for contributors
Zero Trust → Access → Applications → **Add self-hosted app**:
- Application domain: `audio.bedtimetunes.com`, paths `/add` and `/api/add`.
- Policy: Allow → emails (or email domain) of your select contributors.

That's the entire auth layer — no code. Access injects the verified identity; the Worker reads
`Cf-Access-Authenticated-User-Email` and maps it via `CONTRIBUTORS` (unknown → `DEFAULT_UPLOADER`).
Contributors visit `https://audio.bedtimetunes.com/add`, log in once, and submit a Spotify/YouTube
link + title/artist; the Worker parses the ID and inserts the row under their uploader id.

## 6. Deploy the site (Pages)
Pages → Connect to Git → repo. Build command: none. Output dir: `/`. Domain `bedtimetunes.com`.

## 7. Upload MP3s (full playback for the owned ~120)
```powershell
Install-Module AWS.Tools.S3 -Scope CurrentUser
$env:B2_KEY_ID=...; $env:B2_APP_KEY=...; $env:BT_SYNC_TOKEN=...   # = Worker SYNC_TOKEN
.\scripts\upload.ps1
```

## 8. Resolve YouTube IDs (full playback for non-owned tracks)
```bash
pip install yt-dlp
python3 scripts/resolve_youtube.py --base https://audio.bedtimetunes.com --token <SYNC_TOKEN> --limit 100
```
Run in batches. Wrong picks are fixable later by pasting the correct YouTube URL into `/add`.
Tracks with no MP3 and no YouTube id fall back to the Spotify 30s preview + "log in" notice + auto-advance.

## 9. Verify
- `audio.bedtimetunes.com/api/tracks` → full list, owned first.
- `bedtimetunes.com/` → right list + left player; owned tracks start instantly from B2.
- Shuffle toggles in the sidebar; search filters the list.
- `audio.bedtimetunes.com/add` → prompts Access login, then lets you add.

### Source preference (per track, client-side)
`mp3_key` → `youtube_id` → `spotify_id`. The player shows the matching surface: your MP3 player
(with aurora art + full controls), a YouTube embed, or the Spotify preview embed.

### Notes
- Spotify embed only plays full for logged-in Premium visitors; logged-out = 30s. That's why
  YouTube is the preferred full-playback source for non-owned tracks.
- `genre` is ~31% blank and artist-level from Spotify — curate over time.
- Owned tracks whose legacy artist was a film/compilation (e.g. Amelie) won't auto-link a
  spotify_id; fix via `/add` if you want the Spotify fallback on them.
